Карта Now / Next / Later

Каждая карточка — источник-перспектива, impact/effort и зависимость. Цвет источника: Стратег GTM Инженер. Категория действия: ADD добавить · IMPROVE починить/улучшить.

Now

5 пунктов

1.Funnel-инструментирование

Стратег IMPROVE impact высокий S–M

Один funnel-view: signup → onboard → первый тест → D1/D7 → pre-order intent + engagement по механикам. Без этого весь список ниже — гипотезы, а не решения.

Зависимость: нет — события в PostHog частично есть

2.Pre-order → price-discovery

СтратегGTM IMPROVE impact высокий S

Конкретная цена + early-bird на pre-order, разбивка intent по тиру и фиче (band / Writing / Speaking). Единственный revenue-сигнал до merchant-ключей.

Зависимость: ценовой якорь — решение владельца (вопрос №1)

3.Честный empty-state + контент-гейт на пуши

СтратегGTM IMPROVE impact высокий S

Копия «каталог выкатывается [дата]» вместо вида «сломанного» каталога; не постить sprint join-URL в канал до контент-порога — 500 подписчиков не восстановимы.

Зависимость: график перезаливки контента (владелец, вопрос №2)

4.Storage-гигиена: OPS-1 + аудио-кап

Инженер IMPROVE impact высокий S–M

Чистка Storage-сирот (BACKLOG OPS-1) + битрейт-кап на импорте mp3. Supabase Free = 1GB; ~60–120 listening-тестов уже упрутся в лимит.

Зависимость: синхронно с перезаливкой контента клиентом

5.Тихие поломки retention-каналов

Инженер IMPROVE impact средний S

Проверить: weekly-digest роут отсутствует в vercel.json crons — возможно, вообще не срабатывает. Plus: prune signup_throttle, advisory-lock → ON CONFLICT-счётчик до наплыва.

Зависимость: нет

Next

5 пунктов

6.Оффер-репозиционирование: journey + W/S

СтратегGTM IMPROVE impact высокий S–M

Лендинг/pricing вокруг «дистанции до target band к дате экзамена» + W/S громко («AI проверит за минуты, дешевле репетитора»). Обещаем range+confidence, не точный балл — снимает риск невалидированного якоря.

Зависимость: копирайт — решение владельца

7.Student Telegram-бот (text-first)

СтратегGTMИнженер ADD impact высокий M–L

Этап 1: bind-флоу telegram_id↔profile (greenfield, миграция + одноразовый токен). Этап 2: «вопрос дня» + streak/mistakes-due нуджи с deep-link. Без image-карточек и mini-app.

Зависимость: контент + бот-токен (владелец, вопрос №3)

8.Лидерборд-дебаунс

Инженер IMPROVE impact высокий (scale) M

Rebuild лидерборда не чаще раза в ~60с. Сейчас — полный table-lock rebuild на каждый rated submit; спринт-когорта = burst = деградация всех сабмитов разом.

Зависимость: обязателен до п.9

9.Публичный 30-day sprint в канал

GTM ADD impact высокий M

Датированная когорта поверх уже готовых daily-plan + countdown, вехи под шеринг результата.

Зависимость: контент-порог (п.3) + лидерборд-дебаунс (п.8)

10.Fan-out батчи digest/reminders

Инженер IMPROVE impact средний S

Пагинация с курсором вместо одного вызова на всех юзеров — иначе на росте базы таймауты режут список получателей недетерминированно.

Зависимость: до роста базы

11.Платёжные адаптеры Payme/Click/Uzum

Инженер ADD impact высокий M / провайдер

Ядро (applyCompletedPayment) готово и adversarial-проверено. Работа = state-machine протокола каждого провайдера + их sandbox.

Зависимость: merchant-ключи (владелец, вопрос №4) — жёсткий блокер

12.Supabase Pro → Vercel Pro

Инженер IMPROVE impact средний S–M

Триггеры по порядку: Storage 1GB → PITR-потребность → cron/compute-потолок. Supabase сработает первым.

Зависимость: бюджет владельца (вопрос №6)

13.B2B2C-партнёрства с IELTS-центрами

GTM ADD impact высокий M (процесс)

Второй источник трафика с низким CAC — класс-коды для преподавателей, «тренажёр между занятиями».

Зависимость: контент + аутрич владельца (вопрос №7)

14.Лидерборд-витрина + реальная реферал-награда

GTM ADD impact средний S–M

Разморозка соревновательных механик после живого DAU; реферал-награда меняется с XP на free-дни после платежей.

Зависимость: триггер — DAU-сигнал + живые платежи

Останавливаем / не трогаем

Тоже часть ответа «что делать». Здесь — где перспективы разошлись и решение принято явно, плюс фантомный техдолг, который не нужно чинить.

⏸

Лига / лидерборд — заморозка инвестиций DIFFERENTLY

Расхождение: GTM видел здесь виральный актив («топ Ташкента»); стратег — риск: пустой после вайпа лидерборд читается как анти-social-proof, а 4 перекрывающиеся системы удержания на пре-revenue продукте нарушают anti-goal BRIEF §0. Решение: победил стратег — код не удаляется, инвестиции стоп до живого DAU. Дешёвый шеринг результата (уже на проде) остаётся; «витрина хвастовства» — в Later (п.14) с триггером.

⏸

XP-реферал как growth-рычаг

GTM: K-factor ≈ 0, награда без внешней ценности — growth-театр. Оставить только как attribution («кто кого привёл»); вложения в развитие — только после реальной награды (free-дни/скидка) на платежах.

🚫

Фантомный техдолг из списков аудитов

Инженер проверил в коде: rate-limit на login уже реализован (auth.ts:90) — заметка аудита устарела; два exam-раннера (iframe-mock + атомизированный) — осознанная архитектура, не сливать в один; RLS/кэш-постура answer_key — многослойно проверена, «упрощать» = ломать защиту.

🚫

Image band-карточка, Telegram mini-app

Требуют новый runtime-стек (image-рендер / initData-HMAC-авторизация вместо Supabase-сессии) — anti-scope без сильного «почему сейчас». Ценность нуджей достигается текстом + deep-link без них.

Вопросы владельцу

Продуктовые развилки, которые не решаются кодом. Карта выше на них местами прямо завязана.

1

Ценовой якорь pre-order

Какую цену вешаем (допущение GTM: mass-market коридор ~30–100k UZS/мес) и какая early-bird скидка? Дифференцируем по тирам с первого дня?

2

Контент-порог для громкого пуша

Сколько R/L-тестов — минимум, чтобы звать базу в канал на sprint? Ориентировочный график перезаливки.

3

Student-бот: делаем?

Если да — нужен отдельный бот-токен. Это самая крупная кодовая инвестиция карты (M–L).

4

Merchant-онбординг: какой провайдер первым

Payme / Click / Uzum — адаптеры пишутся по одному, порядок определяет ваш реальный онбординг.

5

Подтверждение freeze лиги/лидерборда

Единственное место, где перспективы разошлись явно. Решение выше — но право на него ваше.

6

Бюджет Supabase Pro

Перезаливка каталога может упереться в Storage 1GB в горизонте недель. Платим по триггеру или выносим аудио во внешний бакет заранее?

7

B2B2C-партнёрства — ваш процесс?

Интересен ли трек «тренажёр для учеников IELTS-центров»? Код поддержит класс-кодами, продажи — не кодовая работа.
