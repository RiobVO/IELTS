> Назначение: приоритезированный список продуктовых ходов из живого ревью
> (5 шляп: PM / UX / Growth / Senior Eng / IELTS-методист) — что **добавить**,
> **улучшить**, сделать **иначе**, чтобы продукт стал уровня «100, а не 20».
> Это НЕ операционный чеклист запуска (он в [LAUNCH.md](./LAUNCH.md)) и НЕ спека
> (она в [BRIEF.md](./BRIEF.md)). Источник правды по фазам — `CLAUDE.md §Status`.
>
> **Контракт:** работаем волнами, один пункт = один подтверждённый шаг (как в
> [WORKLOG.md](./WORKLOG.md) §5). Перед началом пункта — план + acceptance, ждать
> «делай». НИЧЕГО не ломать (grading/submit/RLS/tiers/рейтинг идентичны). Стиль:
> inline-стили + токены, ноль новых runtime-зависимостей.

## Как читать

- Статусы: `☐` не начато · `🔄` в работе · `✅` закрыто (дата + 1 строка «что сделано»).
- Категория: `[ADD]` добавить · `[IMPROVE]` улучшить · `[DIFFERENTLY]` сделать иначе/убрать.
- Усилие: `S` (часы) · `M` (день-два) · `L` (отдельная волна).
- Каждый пункт привязан к метрике: activation / retention / revenue (free→paid) /
  виральность / рост балла.
- Провенанс: находки из живого прохода прод-сайта `ielts-rho-seven.vercel.app`
  (2026-06-18), вход Basic-юзером. Не выдумано по коду — увидено вживую, кроме
  явно помеченных гипотез.

## Скоуп-граница (что НЕ в этом файле)

- Деплой / ключи / контент в live БД / Lighthouse-замеры → `LAUNCH.md`.
- **Mobile / responsive** `/app` — известный отложенный долг (`CLAUDE.md`,
  desktop-only). НЕ дублирую как находку, но это **структурный must №1 для
  phone-first UZ-аудитории**: без него все пункты ниже на мобиле умножаются на ноль.
  Делать отдельной волной (L).
