## 0. Verdict

Проект **зрелый и структурно безопасный**: ядро (`answer_key` / RLS / tier-гейты / server-only
grading / anti-cheat / платёжный вебхук) подтверждено чистым **в коде**, а не только в доках.
**P0 не найдено** ни по одной из шести осей. Модель владения данными корректна: все 28 таблиц с
RLS, owner-пути (Drizzle) везде фильтруют по `user_id = session`, «секретные» таблицы (`answer_key`,
`attempt_review_snapshot`, оба `*_feedback_debug`, `error_log`) hard-locked (RLS + REVOKE ALL) и это
**позитивно ассертит `verify.ts`**. Клиентский бандл чист — server-only секреты/промты/ключи в
браузер не утекают. Биометрия (аудио) — приватный бакет, signed URL, owner-scoped storage-policy,
удаление retryable с честным статусом (post-`0032`).

Реальный остаток — **defense-in-depth дыры уровня P1(гранично)/P2, не активные эксплойты**:
(1) `runner_html` основной iframe-архитектуры anon-читаем, а барьер против утечки ключа — только
import-time санитайзер с задокументированными слепыми зонами, без DB-lock и без verify-ассерта;
(2) DNS-rebinding TOCTOU в SSRF-гварде импорта аудио; (3) Speaking-оценка без rate-limit (cost-amp);
(4) при сбое удаления аудио биометрия живёт до 7 дней вместо «следующего прохода». Единственный
**функциональный launch-blocker** — реальные подписи платёжных провайдеров (`#9`/`D1`, generic-HMAC
плейсхолдер): не дыра «доступ бесплатно», а «платежи не проходят». **Готовность к проду: высокая по
безопасности, с оговорками по платежам и по хрупкости anti-key-leak для runner-тестов.**

## 1. Что отлично (сохранить) — подтверждено лично в коде

- **`answer_key` физически недоступен клиенту** — RLS on + `REVOKE ALL FROM anon, authenticated, PUBLIC`
  (`migrations/0001_rls/up.sql:69`); `verify.ts:325` ассертит три слоя (RLS + отсутствие клиент-политики +
  живой anon-SELECT под ролью → 42501).
- **`verify.ts` — настоящий gate**, не «политика есть»: `tableLock` для `answer_key` (:325),
  `attempt_review_snapshot` (:337), обоих `*_feedback_debug` (:349), `error_log` (:377); anon-lock для
  `leaderboard_entry` (:365); write-lockdown (:391) — запрет патча `profile.role` и форжа `submitted`-attempt,
  при рабочем owner-write.
- **Платёжный вебхук** — тело не доверяется (сумма/tier/owner из PENDING-строки + `validateEntitlement`,
  `payments/index.ts:106-160`), идемпотентность через `UNIQUE(provider,tx)` + атомарный claim
  (`:182-198`), STUB отклоняется в prod (`:56-62`), подпись constant-time (`webhook-signature.ts:25`).
- **Owner-пути без IDOR** — все Drizzle-чтения/записи по id из запроса фильтруют `user_id = session`
  (`readFeedbackResult` `writing/read.ts:133`, `readOwnSubmission`, `submitAttempt` SELECT
  `reading/[id]/actions.ts:81`).
- **Tier-гейты на сервере** — `canEvaluate` + per-task `tierRequired` + runner-route `meetsTier`→403;
  free-preview лимитирован 1 lifetime completed.
- **iframe-раннер** — opaque-origin sandbox без `allow-same-origin`, приём сабмита по идентичности окна,
  CSP deny-by-default (подтверждено в `AUDIT_2026-07-01` §1, не регрессировало).
- **Клиентский бандл** — единственные `process.env` в браузере — `NEXT_PUBLIC_*` (`supabase/client.ts:6-7`,
  `instrumentation-client.ts`); промты/`*_feedback_debug`/`answer_key` в клиент не попадают.

## 3. System Map (фактическая)

- **Runtime/deploy**: Next.js 15 App Router (React 19), Vercel Hobby (fra1, Fluid), Supabase Postgres
  (eu-central-1). Оба региона — Франкфурт, сетевой floor ~200 мс — инфра, не app-code.
- **Auth/session**: `@supabase/ssr`, middleware рефрешит сессию через `getClaims()` (локальная
  ES256-верификация, без auth-RTT). Гейты страниц `getUser`/`getProfile` (`cache()`-мемо). `/app` и
  `/admin` защищены в middleware; роль admin доперепроверяется в `requireAdmin` (`src/lib/auth.ts:58`).
- **Два DB-пути**: (1) Supabase anon-client — RLS-scoped, user-writes; (2) Drizzle owner-client (`@/db`,
  `service_role`, обходит RLS) — server-only: grading, import, feedback-чтение (везде owner-check в WHERE).
- **Роли/планы**: `basic<premium<ultra` (`src/lib/tiers.ts`), `effectiveTier` не доверяет протухшему
  tier. Writing=Premium, Speaking=Ultra + 1 lifetime-preview суб-тиру. Гейты дублируются на сервере
  (`canEvaluate`, `enforceAccess`, runner-route `meetsTier`→403).
- **Схема/миграции**: 34 миграции up/down, hand-authored SQL + `_migrations` bookkeeping, host-guard на
  `down`/`bootstrap`. 28 app-таблиц, RLS на всех.
- **Storage**: `speaking-audio` — приватный бакет (`public=false`, 10 МБ, `audio/webm|mp4`), owner-scoped
  storage-policy (path `${userId}/${id}`), signed PUT/GET; политика живёт в
  `scripts/setup-speaking-storage.ts` (**вне цепочки миграций** — drift-риск, N13-соседство).
- **API/actions/webhooks**: 12 route-handlers (health, 4 cron, 2 internal-evaluate, payment-webhook,
  telegram-webhook, runner, auth-callback, client-error), ~10 server-actions. Webhooks/cron/internal
  исключены из middleware-auth, гейтятся собственной подписью/секретом (constant-time, fail-closed).
- **AI-флоу**: async store→internal secret-gated route→poll; Gemini Flash (audio-native для Speaking),
  опц. Whisper для таймкодов. Промты и raw-output строго server-only.
- **Admin/import/debug**: `/admin` (requireAdmin) + Telegram-бот (whitelist по `adminIds` + secret-token).
  Import детерминированный (cheerio + `node:vm` global-free, timeout, 4 МБ gate), без LLM/eval.
  Debug-данные — `error_log` + `*_feedback_debug`, все admin/owner-only.
