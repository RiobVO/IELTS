## Контракт (правила, стоп-условия)

- **Порт раннера = ЧИСТЫЙ ФРОНТ.** БЕЗ новых миграций/схемы/сигнатур server-actions/RLS.
  Аннотации (highlight/notes) уже в БД (`0013_annotation`) — это сделанное исключение, не трогаем.
  Если аффорданс требует новую схему/эндпоинт → СТОП, вынести вопросом.
- **Грейдинг остаётся СЕРВЕРНЫМ.** Референсы держат `KEY`/`correctAnswers` + `isCorrect()`/
  `band()`/`computeReport()` в клиентском JS — **не переносим** (§6.1/§4.6). `answer_key`
  никогда не уходит в клиент. Перенос клиентского ключа = стоп-условие.
- **Не трогаем:** `result/review` (Волна 1: per-type breakdown, open-флаг), каталог/дашборд/
  лигу/бейджи, соседний код. Раннер заканчивается сабмитом → ведёт на существующий bando-результат.
- **Вид:** решение пользователя (2026-06-21) — **Cambridge-скин на оба режима** (шаг 4),
  **шрифты наши** (Jakarta/Literata/JBMono) как бренд-штрих. Только раннер; остальной app — bando.
- React, существующие токены/компоненты, inline-стили. **НОЛЬ новых runtime-зависимостей**
  (`package.json` не трогаем). Mobile/responsive раннера — НЕ в этом треке (не ломать существующий).
- Аудио — из Supabase Storage; premium-аудио — signed URLs.
- Идентификаторы/коммиты по-английски, без AI-атрибуции. Общение по-русски.

## As-is раннера (что уже есть — не дублировать)

- Reading + Listening — **один** раннер `app/app/reading/[id]/` (`ExamRunner.tsx`); Listening
  определяется наличием `passage.audio_path` (`isListening = !!audioSrc`). Отдельного listening-роута нет.
- `PassagePane` переверстан в editorial bando (masthead, drop-cap, буквы абзацев, reading-progress, tool-капсула).
- Highlight/notes **персистятся в БД** (`0013_annotation`, server-actions, `wrapOffsets` на mount) — переживают reload.
- Размер шрифта A−/A+ и темы (paper/sepia) — есть (localStorage, `bando-reading-size`/`bando-reading-theme`).
- Серверный грейдинг (`submitAttempt` читает `answer_key` owner-путём), серверное время от `started_at`,
  autosave (debounce 1500ms), resume (`ensureAttempt`, idempotent), идемпотентный submit — **работают**.
- Флаг review **есть**, но клиентский/эфемерный (не персистится между сессиями) — для трека ок.
- Single-pass: waveform некликабелен (нет seek), **но пауза разрешена** → строгий проход недоделан.
- Таймер — wall-clock (`elapsed` +1s), не привязан к аудио. Transfer-time нет.

## Под-шаги (☐ todo / 🔄 в работе / ✅ done)

1. ✅ **Нижний навигатор 1–40 full-width** — Part-группировка, review-флаги, состояние то же
   (`QuestionNavigator` перепрофилирован side→footer; `passage_id` добавлен в read вопросов).
2. ✅ **Practice/Mock + start-screen** (Reading-only) — overlay выбора режима; Practice (счёт вверх,
   пауза, рестарт) / Mock (обратный отсчёт + авто-сабмит при 0). Mock — клиентский UI, `mode` в БД
   НЕ пишется, `ensureAttempt`/серверное время не тронуты; авто-сабмит зовёт существующий `submitAttempt`.
   Решения по ходу (на ревью):
   - Режим хранится в `localStorage` (keyed по `attemptId`) → refresh не сбрасывает Mock-дедлайн
     (привязан к wall-clock) и **не даёт бесплатный реплей**; resume без режима → Practice без start-screen.
   - Истёкший Mock при возврате (дедлайн в прошлом) → авто-сабмит на загрузке («время вышло»).
   - **Restart** (Practice) = очистка ответов+флагов+таймера за `confirm()` (как в референсе).
   - Клавиша «P» для паузы намеренно НЕ добавлена (чтобы не перехватывать ввод в поля ответов) — пауза кнопкой.
   - Mock-пресеты минут: `[20/40/60 + дефолт]` (дефолт из `duration_seconds`, иначе по числу вопросов).
   - **Listening не тронут** (его таймер/single-pass — шаг 3).
3. ✅ **Listening строгий single-pass + буфер + audio-таймер + transfer-time** (Listening-only).
   - Старт/резюм записи — через **gate-оверлей** (нужен жест: автоплей иначе блокируется браузером);
     буфер-индикатор (% загрузки), Play активна по `canplay`.
   - **Без паузы/seek/replay:** `AudioPlayer` в `locked`-режиме (статус-индикатор вместо play/pause).
   - **Audio-синхронный таймер:** остаток записи привязан к wall-clock якорю первого Play; ≤5 мин warn,
     ≤1 мин critical (как «последняя минута»).
   - **Transfer-time = 2 мин** (Cambridge-симулятор) после конца записи → авто-сабмит при 0
     (зовёт существующий `submitAttempt`; сервер считает время и грейдит).
   - **Refresh = резюм с реальной позиции (forward-seek), без реплея:** якорь старта в `localStorage`
     (по `attemptId`); вернулся в окне записи → gate «Continue» резюмит с текущей точки; вернулся в
     transfer → сразу окно переноса; время вышло → авто-сабмит. **Бесплатного реплея нет.**
   - ⚠️ Логика покрыта `tsc`/`build`/`test`, но **аудио-поведение (seek/autoplay/буфер) надо глянуть
     вживую** на проде с реальным listening-тестом.
4. ✅ **Cambridge-скин раннера** (оба режима, шрифты наши). Подход: один scoped-блок
   `.exam-cambridge` на корне раннера переопределяет дизайн-токены под бело-сине-графитовую
   палитру реального computer-IELTS (`#ffffff` фон, `#cfe0f0`/`#9fc6ea` бордеры, `#2563eb` акцент,
   `#111827` текст, `#f59e0b` флаг, плоские тени) → шапка/навигатор/таймер/карточки/оверлеи/пассаж
   перекрашиваются разом. Пассаж «расхамлен» (без буквицы; буквы абзацев плоские, не кружки).
   **Только CSS-токены + 1 className** — структура/логика/аннотации/грейдинг не тронуты.
5. ✅ **Геометрия/типографика под референс** (после живого ревью: шаг 4 был только перекраской,
   bando-форма осталась — serif-пассаж, скруглённые карточки-вопросы, чипсы, pill-опции). Плоский вид
   как настоящий computer-IELTS: пассаж → **sans** (наш Jakarta через `--font-reading`), заголовок по
   центру, убраны чипсы «words/min read»; вопросы → **плоско** (без карточек-рамок/теней, тонкий
   разделитель снизу), опции компактнее/тоньше (1px), колонка вопросов белая, радиусы площе.
   Только CSS/стили — логика/структура/аннотации не тронуты.
   **Остаётся bando-элемент:** плавающая tool-капсула (highlight/note/A−A+/тема) — в референсе эти
   инструменты в меню шапки; перенос — отдельной правкой, если нужно.

## Открытые риски / решения (зафиксировано 2026-06-21)

- **mode → `ensureAttempt`:** НЕ трогаем. Mock — чисто клиентский UI. Писать `mode='mock'` в БД —
  только если позже понадобится отличать «сдал под экзаменом» в аналитике (отдельная задача).
- **Listening refresh:** резюм с серверной позиции **или форфейт**, не рестарт. Реальный IELTS = один проход.
- **transfer-time длительность (шаг 3):** ✅ принято **2 мин** (Cambridge-sim check-phase).
- **Listening single-pass якорь — клиентский** (`localStorage`), НЕ серверный: трекинг позиции на сервере
  потребовал бы схему/эндпоинт (стоп-условие). Forward-seek к реальной позиции даёт «no replay»; если
  нужна серверная гарантия — отдельная задача (схема/эндпоинт).
